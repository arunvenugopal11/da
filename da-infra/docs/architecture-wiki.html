3. AWS infrastructure

Lean AWS design principle: Four always-on cost sinkholes are eliminated — NAT Gateway, ALB, ECS Fargate, and ElastiCache. Together they save ~$96–126/mo at every stage of growth.
❌ Removed
NAT Gateway$33/mo
ALB$18/mo
ECS Fargate$30–60/mo
ElastiCache$15/mo
✓ Replaced with
VPC Gateway EndpointsFree
API Gateway HTTP v2$1/M req
Lambda (scale to zero)$0 idle
DynamoDB TTL cache$0 idle

VPC topology

VPC: da-{env}-vpc
│
├── Private subnets (2 AZs)
│   ├── Lambdas: auth, profile, matching, premium, chat
│   ├── RDS PostgreSQL
│   └── RDS Proxy (mandatory — prevents Lambda connection exhaustion)
│
└── Public subnets (2 AZs) — reserved for ALB if added later

Outside VPC (no RDS dependency):
└── Lambdas: notifications, stripe_webhook
    └── Reach Expo Push / Stripe via internet directly

Free VPC Gateway Endpoints (no data charge):
├── s3         → Lambdas reach S3 without NAT
└── dynamodb   → Lambdas reach DynamoDB without NAT

Interface Endpoint (small hourly cost):
└── ssm        → Lambdas read Parameter Store without NAT

RDS Proxy is non-negotiable. Without it, 100 concurrent Lambda invocations exhaust the connection limit and every query fails with "too many connections". Proxy costs $18/mo and multiplexes connections — mandatory from day one.

6. Async & eventing

SQS queue pattern

Every queue in this project is created with this structure — enforced by the Terraform module:

Primary queue
├── maxReceiveCount: 3       → retry 3× before DLQ
├── Deny non-SSL (policy)    → all traffic must use TLS
└── DLQ (7-day retention)
    └── CloudWatch alarm     → fires when any message lands in DLQ
QueueProducerConsumerPurpose
da-{env}-notificationmatching, premium, chatnotifications LambdaPush notification fan-out
da-{env}-matchingprofile Lambdamatching LambdaAsync match score computation

EventBridge Scheduler

Daily cron at 02:00 UTC triggers matching Lambda to pre-compute recommendations for all active users → writes to da-{env}-daily-recommendations DynamoDB table with 24hr TTL.

7. Security model

IAM permission boundaries

Every IAM role has permissions_boundary = Da-{env}-PermissionBoundary. Even a wildcard Action:"*" role cannot exceed what the boundary allows.

Explicitly denied regardless of role policy:

  • iam:CreateRole, iam:AttachRolePolicy, iam:PutRolePolicy
  • iam:DeleteRolePolicy, iam:DetachRolePolicy
  • organizations:*, account:*

Secret flow

HCP Vault Secrets (free tier)
↓ fetched at deploy time
GitHub Actions TF_VAR_* (masked)
↓ written by Terraform
SSM SecureString (KMS encrypted)
↓ read at Lambda cold start
Application code (never logged)

Network controls

Security groupRule
Lambda SGEgress TCP 443 → VPC CIDR (SSM, S3, DynamoDB endpoints)
Egress TCP 5432 → VPC CIDR (RDS Proxy)
RDS SGIngress TCP 5432 from Lambda SG only — no other inbound
Lambdas connect to RDS via RDS Proxy using IAM authentication — no database password in code or environment variables. The proxy authenticates via rds-db:connect IAM permission scoped to the proxy resource ARN.

The prod manual gate uses GitHub Environment protection rules — go to Settings → Environments → prod → Required reviewers and add yourself. The workflow pauses until you click "Approve and deploy" in the Actions UI.

AWS authentication

No static IAM access keys in GitHub. GitHub Actions authenticates via OIDC — assumes a role using a short-lived token. The IAM role trusts token.actions.githubusercontent.com as the identity provider. Zero long-lived credentials stored anywhere.

11. Architecture decisions

DecisionChoiceRationale
ComputeLambda arm64Scale to zero, Graviton2 20% cheaper, faster cold starts
API routingAPI GW HTTP v2$1/million vs ALB $18/mo minimum
ChatAPI GW WebSocket + LambdaNo ECS; 2hr timeout handled by client auto-reconnect
DatabaseRDS PostgreSQL + PostGISComplex queries, geo support, multi-condition filters — NoSQL can't do this
CacheDynamoDB TTLShared cache at near-zero cost, no always-on instance
AuthCognito → custom JWT at 80k MAUCognito free to 50k MAU; custom JWT saves $275/mo at scale
MediaS3 + CloudFront OACPrivate bucket, CDN delivery, presigned upload URLs
IaCTerraform ≥ 1.5Enterprise module pattern, remote state, reproducible envs
SecretsHCP Vault SecretsFree tier, clean CI/CD integration, never in state files
IAMPermission boundaries on all rolesPrevents privilege escalation even from misconfigured roles
NotificationsExpo Push SDKAbstracts FCM + APNs, free, no SNS mobile push cost

What never changes regardless of scale: Lambda function code, DynamoDB table structure, S3 bucket config, API Gateway routing rules, Terraform module structure. You scale by changing config and instance sizes — not by rewriting services.
